# Competency Map

## Levels

- **1** — Foundations. Has the basics. Following tutorials, defaults, and first principles.
- **2** — Functional. Gets the job done reliably. Follows known patterns, catches common issues.
- **3** — Deliberate. Conscious choices in context. Systematic, consistent, and intentional.
- **4** — Strategic. Judgment under ambiguity. Sees the whole system. Shapes the environment.
- **5** — Mastery. Unconscious competence. Innovates in the domain. Teaches others at this level.

Assessment uses "best fit" — the level whose description most closely matches observations, not "must meet ALL criteria." Users can be at different levels across domains.

## Resilience & Direction

Staying in the game long enough to win. Energy, pace, identity, direction.

1. Has a project. Works on it. Basic consistency.
2. Clear direction. Regular cadence. Recovers from setbacks without spiraling.
3. Evidence-based pivoting. Sustainable pace. Manages energy, not just time. Knows when to cut scope.
4. Persist vs. quit judgment. Navigates the messy middle. Long-term vision survives tactical changes.
5. Identity as builder, not as tools or job title. Mentoring others. Building systems that survive your absence.

## Product & Design

Building the right thing and making it look right. Taste, judgment, problem framing. Defining what to build and why before touching any tool.

1. Basic UX. Logical flows. Something exists that a user can interact with.
2. Visual consistency. Clear value prop. Coherent information architecture. Knows who the user is.
3. User research (talking, not guessing). Design systems. Accessibility. Specs before code. Iterating from feedback.
4. PMF judgment. Competitive positioning. Feature prioritization. Differentiating taste that others notice.
5. Defining new categories. Design as strategy. Product judgment that others learn from.

## AI Steering

How to get AI to do what you want, and when not to use it. Includes learning new tools quickly, evaluating before adopting, and managing code you didn't write.

1. Basic prompting. Chat-based AI tools. Accepting output mostly as-is.
2. Structured prompts. Context files. Knowing what to accept vs. reject. Catching obvious errors.
3. Context engineering. Spec-driven development. Evaluating and adopting new tools. Right tool for the task. Managing comprehension debt.
4. Agent orchestration. Agent-friendly codebases. Custom toolchains. Knowing when NOT to use AI.
5. Designing AI systems for others. Novel interaction patterns. Teaching AI workflows at scale.

## Building & Architecture

Making things that work, hold together, and don't collapse under load. Seeing how parts connect. Knowing when something is right vs. when it merely looks right.

1. Ship a working app. Basic CRUD. Deploy. Git.
2. Clean file organization. Consistent patterns. Basic error handling. Version control discipline.
3. Testing discipline. CI/CD. Coherent architecture across files. Deliberate data modeling.
4. System design for scale. Observability. Cost-aware architecture. Performance optimization under constraints.
5. Designing systems others build on. Technical leadership patterns. Architecture that teaches.

## Distribution & Growth

Getting people to use what you built. Day-one concern, not afterthought.

1. Has a landing page. Product exists on the internet. Someone besides you knows about it.
2. Clear value prop. Basic SEO. Social presence. A few strangers have used it.
3. Content strategy. Channel focus. Manual-first validation. Building an audience. Pricing experiments.
4. Distribution as moat. Network effects. Community building. Repeatable acquisition channels.
5. Distribution innovation. Creating new channels. Growth systems that compound without you.

## Security & Reliability

Not getting hacked. Not losing data. Not leaking secrets.

1. No secrets in code. Env vars. Basic auth. HTTPS.
2. Input validation. Dependency awareness. Rate limiting basics. Auth flows that work.
3. OWASP Top 10 awareness. CSRF/XSS prevention. Dependency scanning. Incident response plan.
4. Threat modeling. Security architecture. AI-specific security (prompt injection, data poisoning). Defense in depth.
5. Security as product feature. Proactive vulnerability research. Building security culture.

## Business & Operations

Money, legal, support, systems. The gap between a shipped MVP and a sustainable business.

1. Revenue exists or is planned. Basic legal (terms, privacy). Payments work.
2. Costs tracked. Support exists. Basic analytics. Pricing that someone has paid.
3. Unit economics (CAC, LTV, churn). Ops automation. Systematic feedback loops. Repeatable revenue.
4. Scaling beyond yourself. Hiring/delegation. Financial modeling. Systems that run without you.
5. Business model innovation. Strategic partnerships. Building organizations, not just products.
